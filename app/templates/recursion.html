In the previous lecture, we implemented binary search by splitting a list into two halves and binary searching each halve. In computer science, this process is known as “recursion”. More formally, recursion is a function calling itself in order to solve a larger problem.

In general, recursive problems have a base case and a recursive case. The base case is a case at which the problem is done or at which we don’t need to call the recursive function anymore. The recursive case is the opposite. It is when we still have to call the recursive case more in order to solve the problem. Let’s look at a few examples.

Suppose we want to calculate the factorial of a number, n. The factorial is denoted n! and is defined as n! = n x (n-1) x (n-2) x … x 3 x 2 x 1. Now, convince yourself that n! = n x (n-1)! is a valid equation. Once, we write this, we have turned the problem into a simpler problem that only involved finding (n-1)!. Try this next one on your own! The Fibonacci numbers are defined as f(n) = f(n-1) + f(n-2). Also, it is defined that f(1) = 1 and f(0) = 1. The first couple elements of the Fibonacci sequence are:

f(0) = 0
f(1) = 1
f(2) = f(1) + f(0) = 1
f(3) = f(2) + f(1) = 2
f(4) = f(3) + f(2) = 3
and so on!
