Let’s say we want to use Python to model a cat named Pycat. We want to Pycat to have a name and an age. Additionally, we want Pycat to know how to say “Meow”.

Python allows us to create Pycat by using what programmers call a “class”. Pycat’s name and age are known as “attributes” - they are variables that are contained within the Pycat class. Having Pycat saw “Meow” is known as a “behavior” - it is a function that the Pycat class can perform. In Python, we define a class as follows:

Let’s break this code down. “class Pycat” is the command for defining a new class named Pycat. “def” defines a new function that Pycat can do. However, “__init__” is a special function. This tells us that when we define a new Pycat, we need to provide Pycat’s name and age. You might be wondering, why would we want to create classes? Classes allow us to group similar functions into one entity. Additionally, it gives us access to very important features of Object-Oriented Languages like Python including polymorphism, inheritance, and encapsulation.
